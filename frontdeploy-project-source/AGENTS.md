Project rules for Codex

Commit every meaningful code change immediately.
Never continue to the next feature with uncommitted changes.
Always run git status before and after each commit.
Keep commits small and focused.
Do not include secrets, API keys, private keys, seed phrases, or .env files.
Do not build trading execution, copy trading, or wallet private key handling.
This project is a read-only intelligence overlay for Axiom.
Use mock data first.
Keep architecture ready for future APIs like Helius, Birdeye, Jupiter, and custom backend.
Before final response, run npm run build and commit any fixes.
